## 🔬 Advanced Optimization Tips

### ⚡ The `short` type

Contrary to popular belief, `short` is a **16-bit integer**.

- **Use case:** When you need to store millions of small numbers (like raw 16-bit audio samples) and want to save 50% of memory compared to a standard `int`.
- **Range:** -32,768 to 32,767.

```cpp
short audioSample = 32000; // 2 bytes vs int's 4 bytes — saves memory at scale
```

### 🛡️ The `const` Modifier

If a value is not going to change (like a fixed API endpoint or a math constant), always mark it as `const`.

```cpp
const double HOURLY_RATE = 15.0; // Locked value — compiler will error if you try to change it
const string API_URL = "https://api.elevenlabs.io/v1"; // Can't be overwritten
```

> [!NOTE]
> `const` is not just style — it's a **contract**. It tells the compiler and other developers: *"this value will never change."* The compiler can then optimize around it.

<h1 align="center">🏗️ Structs vs Classes</h1>

<p align="center"><i>Both create custom data types — but they serve different purposes.</i></p>

<p align="center">
  <img src="https://img.shields.io/badge/Topic-OOP_Basics-8B5CF6?style=for-the-badge&logo=c%2B%2B&logoColor=white" />
</p>

<div align="center">

|  | `struct` | `class` |
| :---: | :--- | :--- |
| 🎯 **Purpose** | Group data together | Data + logic + protection |
| 🔒 **Default access** | `public` | `private` |
| 🧠 **Has memory?** | No — just a snapshot | Yes — remembers state |
| ⚙️ **Has methods?** | Rarely | Always |
| 📦 **Use when** | Simple data bundle (like a business card) | Living object with behavior |

</div>

```cpp
// struct: just a data card
struct Intern {
    string name;
    double rate;
};

// class: a living object with memory and protection
class AudioProcessor {
private:
    double totalRevenue = 0.0; // Nobody can touch this from outside

public:
    void processRequest(double price) {
        totalRevenue += price; // Remembers the running total
    }

    void showReport() {
        cout << "Total: " << totalRevenue << endl;
    }
};
```

> [!IMPORTANT]
> The key difference: a `struct` forgets everything the moment it's read. A `class` with `private` state **accumulates history** between calls.

<h1 align="center">🔐 Encapsulation — Private & Public</h1>

<p align="center">
  <img src="https://img.shields.io/badge/Concept-Encapsulation-EF4444?style=for-the-badge" />
  <img src="https://img.shields.io/badge/Pattern-Data_Protection-F59E0B?style=for-the-badge" />
</p>

Think of it like a phone: you press the screen (public interface), but you can't directly rewire the battery (private internals).

```
┌─────────────────────────────────────┐
│           AudioProcessor            │
│                                     │
│  🔒 PRIVATE                         │
│  ├── double totalRevenue            │
│  ├── Intern client                  │
│  └── Logger history                 │
│                                     │
│  🔓 PUBLIC (your buttons)           │
│  ├── processRequest(text, price)    │
│  ├── setClient(client)              │
│  └── showReport()                   │
└─────────────────────────────────────┘
```

```cpp
class BankAccount {
private:
    double balance = 0.0; // No one can do: account.balance = -99999

public:
    void deposit(double amount) {
        if (amount > 0) balance += amount; // Validated before touching private data
    }

    double getBalance() { return balance; } // Read-only access
};
```

<h1 align="center">📋 std::vector — Dynamic Arrays</h1>

<p align="center">
  <img src="https://img.shields.io/badge/STL-std%3A%3Avector-00599C?style=for-the-badge&logo=c%2B%2B&logoColor=white" />
</p>

A regular array `int arr[10]` has a **fixed size** — you can't fit an 11th element. A `vector` grows automatically as you add data.

```cpp
#include <vector>

vector<string> logs;             // Empty, but ready to grow
logs.push_back("Request 1");     // Adds to the end, expands memory
logs.push_back("Request 2");
logs.push_back("Request 3");

cout << logs.size() << endl;     // Output: 3

// Always use size_t for loop index — matches .size() type
for (size_t i = 0; i < logs.size(); i++) {
    cout << logs[i] << endl;
}
```

> [!TIP]
> Use `vector` whenever you **don't know in advance** how much data is coming — like a list of API logs, audio frequency samples, or incoming requests.

<h1 align="center">⚡ References (&) and `this` Pointer</h1>

<p align="center">
  <img src="https://img.shields.io/badge/Concept-Memory_Efficiency-22C55E?style=for-the-badge" />
  <img src="https://img.shields.io/badge/Concept-this_Pointer-8B5CF6?style=for-the-badge" />
</p>

### 📌 Reference `&` — work with the original, not a copy

```cpp
// ❌ WITHOUT & — makes a full copy of the vector (slow for large audio data)
void normalizeAudio(vector<double> audio) { ... }

// ✅ WITH & — works directly on the original (fast, no memory duplication)
void normalizeAudio(vector<double> &audio) { ... }
```

Imagine your audio vector has **1 million samples**. Without `&`, C++ silently copies all of it into a new variable every time you call the function. With `&`, it just passes a pointer — **no copy, no wasted memory**.

### 📌 `this` pointer — disambiguate class fields from arguments

```cpp
void setClient(Intern client) {
    this->client = client;
    //  ↑ class field    ↑ function argument
}
```

`this` tells the compiler: *"I mean the `client` that belongs to **this specific object**, not the one passed as a parameter."*

<h1 align="center">🧩 Composition — Objects Inside Objects</h1>

<p align="center">
  <img src="https://img.shields.io/badge/Pattern-Composition-F59E0B?style=for-the-badge" />
  <img src="https://img.shields.io/badge/Design-Modular_Code-EF4444?style=for-the-badge" />
</p>

Instead of one giant class with 1000 lines, you build **small, reliable tools** and compose them together — like Lego.

```
AudioProcessor
├── 📋 Logger history       ← separate class, handles all logging
├── 👤 Intern client        ← struct, holds client data
└── 💰 double totalRevenue  ← primitive, tracks revenue
```

```cpp
class AudioProcessor {
private:
    Intern client;      // ← composition: a struct lives inside the class
    Logger history;     // ← composition: a whole other class lives inside
    double totalRevenue = 0.0;

public:
    void processRequest(string text, double price) {
        history.addLog(text);      // Delegate logging to Logger
        totalRevenue += price;
    }
};
```

> [!NOTE]
> When `AudioProcessor` is created, `Logger`'s constructor fires automatically — because `history` is a class field. That's why you saw `"System Logger Started..."` print without calling anything explicitly.
